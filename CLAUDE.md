## Бизнес-контекст

### Проблема
Нужно красивое цифровое приглашение на свадьбу вместо бумажного. Каждый гость получает именную ссылку, где его имя отображается прямо на сайте.

### Пользователи
- Гости свадьбы (~50–150 человек)
- Получают ссылку вида `site.com/?guest=Мария` — видят своё имя в приветствии
- Сценарий: открыть на телефоне, прочитать детали, подтвердить присутствие

### Критерии успеха
- Сайт корректно открывается на мобильных устройствах
- Все гости подтвердили присутствие через форму
- Имя гостя корректно подставляется из URL-параметра
- Сайт выглядит так же красиво, как референсы (Bridgerton, пастельная флористика)

## Дизайн

### Стиль
Вдохновение: сериал **Bridgerton** — романтика, английский сад, много флористики, пастельная палитра.

### Цветовая палитра
```css
--color-lavender: #CFC6E3;  /* лавандовый */
--color-yellow:   #FEF2AA;  /* пастельный жёлтый */
--color-peach:    #FFD599;  /* персиковый */
--color-pink:     #F5CDD8;  /* нежно-розовый */
--color-green:    #C2CB94;  /* оливково-зелёный */
--color-blue:     #A6C2E9;  /* пудровый голубой */
--color-cream:    #FAF5F0;  /* фон — сливочно-белый */
--color-dark:     #4A3728;  /* тёмно-коричневый для текста */
```
Это официальная палитра свадьбы — используется и в дресс-коде, и в декоре. Все UI-элементы, кружки дресс-кода и акценты берутся строго из этих цветов.

### Типографика
- Заголовки: шрифт с засечками в стиле эпохи Регентства (например, `Cormorant Garamond`, `Playfair Display` или `IM Fell English`)
- Подзаголовки/акценты: рукописный курсив (например, `Great Vibes`, `Dancing Script`)
- Основной текст: легкий с засечками или мягкий sans-serif (`Lora`, `Jost`)
- Подключать через Google Fonts

### Референсы

- `reference_3.jpg` — свадебный mood board, пастельная палитра

## Технический стек

### Frontend
- **Технология:** Чистый HTML5 + CSS3 + Vanilla JS (один файл `index.html` + assets)
- **Подход:** Static site, без сборщиков и фреймворков — максимальная простота деплоя
- **Адаптивность:** Mobile-first, breakpoints: 375px / 768px / 1200px

### RSVP
- **Метод:** Formspree — кастомная HTML-форма, данные отправляются на email
- **Endpoint:** `https://formspree.io/f/mpqkpybl`
- **Ответы приходят на:** k.malyshkina@espada.ru
- **Поля формы:**
  - Приду / Не смогу прийти (radio, обязательное)
  - Предпочтения по алкоголю (чекбоксы: вино, шампанское, крепкий, не пью)
  - Аллергии или ограничения по питанию (текстовое поле)
  - Имя гостя (скрытое поле `hidden`, заполняется автоматически из `?guest=` параметра)
- **Преимущество:** Форма полностью стилизована под дизайн сайта (шрифты, цвета, анимации)

### Персонализированные тексты для гостей
- Claude Code генерирует **80 уникальных вариантов** приветственного текста (массив в JS)
- При загрузке страницы текст выбирается детерминированно по хэшу имени гостя из `?guest=` — один гость всегда видит один и тот же текст
- Если параметр `?guest=` не задан — показывается нейтральный вариант без имени
- Тон текстов: романтичный, тёплый, немного поэтичный — в духе Bridgerton
- Примеры вариантов (Claude Code придумывает все 80):
  - *"Анна, этот день будет наполнен любовью — и мы хотим разделить его именно с вами"*
  - *"Михаил, некоторые моменты остаются с нами навсегда. Приходите разделить один из них"*
### Анимации
- **Падающие лепестки:** CSS-анимация + JS (div-элементы с рандомными позициями)
- **Countdown:** Vanilla JS, обновление каждую секунду
- **Scroll-анимации:** Intersection Observer API (появление блоков)
- **Плавный скролл:** `scroll-behavior: smooth` + JS для плавного якорного скролла

### Инфраструктура
- **Хостинг:** Vercel или Netlify (рекомендуется — бесплатно, drag-and-drop деплой)
- **Домен:** Опционально (например, `alexey-karina.ru`)
- **Деплой:** Достаточно загрузить папку с файлами

## Принятые решения (ADR)

### Апрель 2026 — 80 уникальных текстов вместо одного шаблона
**Контекст:** Хочется, чтобы каждый гость чувствовал себя особенным, а не видел одинаковую фразу
**Решение:** Массив из 80 уникальных приветственных текстов в JS; текст выбирается детерминированно по хэшу имени гостя
**Альтернативы:** Один шаблон с подстановкой имени (менее персонально), генерация через AI API в реальном времени (требует бэкенд)
**Последствия:** Один гость всегда видит один и тот же текст; тексты заготовлены заранее Claude Code

### Апрель 2026 — Formspree вместо Google Forms
**Контекст:** Google Forms нельзя стилизовать — iframe всегда выглядит как стандартная форма Google
**Решение:** Formspree — кастомная HTML-форма с полным контролем над дизайном, данные летят на email
**Альтернативы:** Tally.so (частичная кастомизация), Google Sheets через Apps Script (сложнее в настройке)
**Последствия:** Форма выглядит как часть сайта; бесплатный план Formspree — 50 ответов/месяц, для свадьбы достаточно

### Апрель 2026 — Чистый HTML вместо React/Next.js
**Контекст:** Нужен красивый сайт без сложностей деплоя
**Решение:** Статический HTML/CSS/JS без сборщиков
**Альтернативы:** Next.js (избыточно для одной страницы), Webflow (нет контроля над кодом)
**Последствия:** Максимально простой деплой, легко редактировать вручную

### Апрель 2026 — Именные ссылки через URL-параметр
**Контекст:** Хочется персонализации без базы данных
**Решение:** `?guest=Имя` — JS читает параметр и подставляет в DOM
**Альтернативы:** БД + уникальные токены (избыточно), статические HTML-страницы на каждого гостя (неудобно)
**Последствия:** Нет защиты (любой может написать любое имя) — для свадьбы это не критично
